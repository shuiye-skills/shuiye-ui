## 服务器选择

按以下顺序选择：

1. 项目已有的 `package.json` 脚本或框架开发服务器。
2. 环境中已安装的 `live-server`、`serve` 或等价工具。
3. 已有 Node.js 时使用临时的 `npx` 工具，但下载依赖前需要网络和权限确认。
4. 纯静态原型可使用 `python3 -m http.server <port> --directory prototype`。

不要仅因旧文档提到某工具就安装全局依赖。Figma CLI、BrowserSync 和云预览不是基础要求。

## 启动与验证

启动前：

- 确认 `prototype/` 存在。
- 找到 `index.html` 或项目实际入口。
- 检查命令工作目录和资源根路径。
- 选择未占用端口。

启动后：

- 保留进程或终端会话，不要让命令立即退出。
- 使用 `curl` 或等价 HTTP 请求检查入口状态。
- 检查关键 CSS、JavaScript 和图片返回成功。
- 报告准确 URL、端口、服务进程和停止方法。

如果服务器只能绑定 `0.0.0.0`，说明暴露范围；仅需本机时优先绑定 `127.0.0.1`。

## 端口和进程

- 指定端口已占用时，先判断是否为本项目已有服务。
- 不终止未知进程。
- 可选择新端口时，报告变更后的地址。
- 需要停止本次启动的服务时，只停止已确认属于本次任务的进程。
- 后台进程启动后应再次确认其仍在运行。

## 常见问题

### 入口返回 404

- 检查服务器根目录是否为 `prototype/`。
- 检查入口文件名和大小写。
- 单页应用需要 history fallback 时，使用支持该能力的服务器。

### 页面跳转失败

- 检查链接相对当前页面还是相对站点根目录。
- 检查目标文件和大小写。
- 避免把本机绝对路径写入 HTML。

### CSS 或 JavaScript 未加载

- 在浏览器网络面板或 HTTP 请求中检查资源状态。
- 从当前 HTML 文件位置重新计算相对路径。
- 检查 `<link>`、`<script>` 和模块类型是否正确。
- 检查 MIME 类型和跨域限制。

### 直接打开 HTML 正常，HTTP 预览异常

- 检查以 `/` 开头的资源路径是否依赖错误站点根目录。
- 检查模块脚本、fetch 和路由行为。
- 查看控制台和网络请求，不凭视觉猜测原因。

### 端口被占用

- 查询占用进程及命令。
- 如果是现有项目服务，优先复用。
- 如果来源不明，选择其他端口而不是直接终止。

### 页面空白或交互无效

- 查看控制台首个 JavaScript 错误。
- 检查脚本加载顺序和 DOM 查询结果。
- 检查事件绑定是否在 DOM 就绪后执行。
- 确认模拟数据格式和选择器与页面一致。

### 浏览器未自动打开

- 自动打开不是服务器成功的必要条件。
- 先验证 HTTP 地址，再在获得 GUI 权限后打开浏览器。
- 在无 GUI 环境中报告可访问地址和验证结果。

## 安装边界

安装前明确：

- 为什么现有工具不能完成任务。
- 要安装的包、作用域和版本。
- 是否需要网络、全局写权限或令牌。
- 安装后如何验证以及如何移除。

全局 npm 安装、浏览器扩展、Figma 令牌和外部服务连接都需要用户确认。不得把访问令牌写入原型仓库。
